## 1. Sistema de reservas

Crear un sistema de reservas completo para citas de psicología.

* Mostrar un calendario moderno e intuitivo.
* Mostrar únicamente las fechas y horas disponibles.
* No permitir seleccionar horarios ocupados o bloqueados.
* Comprobar la disponibilidad antes de confirmar una reserva para evitar conflictos si dos personas intentan reservar el mismo horario simultáneamente.
* Una vez confirmada una reserva, ese horario deberá dejar de estar disponible automáticamente.

Cada reserva deberá almacenar como mínimo:

* Nombre
* Apellidos
* Correo electrónico
* Teléfono
* Fecha
* Hora
* Duración de la sesión
* Estado de la reserva
* Fecha de creación
* Identificador único

---

## 2. Pago online

Integrar Stripe Checkout.

El flujo debe ser el siguiente:

1. El usuario selecciona una fecha y una hora.
2. Se verifica nuevamente que la hora sigue disponible.
3. Se crea una sesión de pago mediante Stripe.
4. El usuario realiza el pago.
5. Solo cuando Stripe confirme correctamente el pago se almacenará definitivamente la reserva.
6. Si el usuario cancela el pago o este falla, la reserva no deberá confirmarse.

Gestionar correctamente:

* Pago correcto.
* Pago cancelado.
* Pago fallido.
* Abandono del proceso de pago.

## 5. Sistema de reseñas

Crear un sistema seguro sin necesidad de que los pacientes creen una cuenta ni inicien sesión.

El funcionamiento deberá ser el siguiente:

* Solo podrán dejar una reseña los pacientes cuya cita haya sido marcada como "Completada".
* Cuando la psicóloga marque una cita como completada desde el panel de administración, el sistema generará automáticamente un token único, aleatorio y seguro asociado únicamente a esa cita.
* El sistema enviará automáticamente un correo electrónico al paciente con un enlace para escribir su reseña.
* El enlace contendrá ese token seguro.
* Al abrir el enlace el sistema deberá comprobar:

  * Que el token existe.
  * Que pertenece a una cita completada.
  * Que no ha expirado.
  * Que todavía no ha sido utilizado.
* Si todas las comprobaciones son correctas se mostrará el formulario para escribir la reseña.
* Después de enviar la reseña el token quedará invalidado automáticamente.
* Cada cita solo podrá generar una única reseña.
* Las reseñas deberán poder valorarse entre 1 y 5 estrellas.
* También podrán incluir un comentario.
* Las reseñas deberán mostrarse públicamente únicamente después de ser aprobadas desde el panel de administración.
